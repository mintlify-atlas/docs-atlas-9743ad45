# Quick start

This guide will get you up and running with Findex quickly. You'll learn how to launch applications, customize the appearance, and configure basic settings.

<Info>
  Make sure you've completed the [installation](/installation) before proceeding.
</Info>

## Launching Findex

<Steps>
  <Step title="Trigger Findex">
    Use your configured keyboard shortcut or run:
    ```bash
    echo 1 > ~/.config/findex/toggle_file
    ```
    
    The Findex window will appear on your screen.
  </Step>

  <Step title="Search for an application">
    Start typing the name of an application. Findex uses fuzzy search, so you don't need to type the exact name.
    
    For example, typing `ffox` will match "Firefox".
  </Step>

  <Step title="Launch an application">
    Press `Enter` to launch the selected application, or use number keys (`1-9, 0`) to launch one of the first 10 results directly.
    
    - `Enter`: Launch the highlighted result
    - `1`: Launch the first result
    - `2`: Launch the second result
    - And so on...
  </Step>
</Steps>

## Basic usage

### Fuzzy search

Findex uses intelligent fuzzy matching to find applications:

- **Partial matches**: `chr` matches "Google Chrome"
- **Out of order**: `fldr` matches "File Roller"
- **Abbreviations**: `gmp` matches "GNOME Music Player"

### Keyboard navigation

- `Up/Down arrows`: Navigate through results
- `Enter`: Launch selected application
- `Esc`: Close Findex
- `1-9, 0`: Launch the corresponding result directly

### Window behavior

By default, Findex closes when:
- You launch an application
- You press `Esc`
- The window loses focus (configurable)

## Customization

One of Findex's strengths is its deep customization options. Let's explore basic customization.

### Configuration files

On first run, Findex creates two configuration files in `~/.config/findex/`:

- `settings.toml`: Behavioral settings
- `style.css`: Visual styling

### Customizing appearance with CSS

<Steps>
  <Step title="Open the style file">
    ```bash
    nano ~/.config/findex/style.css
    ```
    
    Or use your preferred text editor.
  </Step>

  <Step title="Customize CSS classes">
    Findex exposes several CSS classes for styling. Here's a simple example to change the window background and text colors:
    
    ```css
    /* Main window background */
    .findex-window {
      background-color: #1e1e2e;
    }

    /* Search box */
    .findex-query {
      background-color: #313244;
      color: #cdd6f4;
      font-size: 18px;
      padding: 12px;
      border-radius: 8px;
    }

    /* Results container */
    .findex-results {
      background-color: #1e1e2e;
    }

    /* Individual result row */
    .findex-result-row {
      padding: 8px;
      border-radius: 6px;
    }

    .findex-result-row:selected {
      background-color: #45475a;
    }

    /* Application name */
    .findex-result-app-name {
      color: #cdd6f4;
      font-weight: bold;
      font-size: 14px;
    }

    /* Application description */
    .findex-result-app-description {
      color: #9399b2;
      font-size: 12px;
    }
    ```
  </Step>

  <Step title="Reload Findex">
    Changes take effect immediately. Close and reopen Findex to see your changes:
    ```bash
    echo 1 > ~/.config/findex/toggle_file
    ```
  </Step>
</Steps>

### Available CSS classes

Findex provides these CSS classes for customization:

| CSS Class | Element |
|-----------|----------|
| `.findex-window` | Top level window |
| `.findex-container` | Top level container |
| `.findex-query` | Search input box |
| `.findex-query-container` | Search box container |
| `.findex-results-scroll` | Scrollable results container |
| `.findex-results` | Results listbox |
| `.findex-result-row` | Individual result row |
| `.findex-result-icon` | Application icon |
| `.findex-result-icon-container` | Icon container |
| `.findex-result-app-name` | Application name |
| `.findex-result-app-description` | Application description |
| `.findex-result-app-command` | Application command |
| `.findex-result-trigger-shortcut` | Keyboard shortcut indicator |
| `.findex-result-info-container` | Result info container |

### Configuring behavior

<Steps>
  <Step title="Open settings file">
    ```bash
    nano ~/.config/findex/settings.toml
    ```
  </Step>

  <Step title="Common settings">
    Here are some useful settings you can configure:
    
    ```toml
    # Window appearance
    default_window_width = 600
    min_content_height = 400
    max_content_height = 600
    decorate_window = false
    icon_size = 48

    # Search behavior
    result_size = 10
    min_score = 50
    name_match_highlight_color = "#89b4fa"

    # Window behavior
    close_window_on_losing_focus = true
    query_placeholder = "Search applications..."
    entry_icon = "search"

    # Keyboard shortcut (X11 only)
    toggle_key = "<Alt>space"

    # Monitor selection (zero-indexed)
    on_monitor = 0
    ```
  </Step>

  <Step title="Save and reload">
    Save the file. The daemon will automatically reload the configuration. Close and reopen Findex to see changes.
  </Step>
</Steps>

### Key settings explained

- `result_size`: Maximum number of search results to display
- `min_score`: Minimum fuzzy match score (0-100) for results to appear
- `close_window_on_losing_focus`: Whether Findex closes when clicking outside
- `toggle_key`: Global hotkey to show/hide Findex (X11 only)
- `decorate_window`: Show window title bar and decorations

## Using plugins

Findex supports plugins to extend functionality. Here's how to use a plugin:

<Steps>
  <Step title="Get a plugin">
    Download or build a Findex plugin. Plugins are `.so` files compiled from Rust.
    
    Example plugins available in the Findex repository:
    - URL opener
    - GitHub repository search
    - Command executor
    - Calculator
  </Step>

  <Step title="Configure the plugin">
    Add the plugin to your `settings.toml`:
    
    ```toml
    [plugins.calculator]
    path = "/path/to/plugin.so"
    prefix = "calc!"
    keyboard_shortcut = "<Ctrl><Shift>C"
    config = {}
    ```
  </Step>

  <Step title="Use the plugin">
    Trigger the plugin by:
    - Typing the prefix in Findex: `calc!`
    - Using the keyboard shortcut: `Ctrl+Shift+C`
  </Step>
</Steps>

<Note>
  Always ensure plugins are compatible with your Findex version. Check the plugin documentation for details.
</Note>

## Tips and tricks

### Performance tuning

- Lower `result_size` if you have many applications installed
- Increase `min_score` to show only closer matches
- Adjust `max_content_height` based on your screen size

### Keyboard efficiency

- Use number keys (`1-9, 0`) to instantly launch top results
- Set up plugin keyboard shortcuts for quick access
- Keep `close_window_on_losing_focus = true` for a cleaner workflow

### Visual consistency

Match Findex to your desktop theme by:
- Using your terminal color scheme in the CSS
- Matching border radius values to other applications
- Adjusting font sizes to match your system fonts

## Troubleshooting

### Findex doesn't appear

1. Check if daemon is running: `ps aux | grep findex-daemon`
2. Start daemon manually: `findex-daemon`
3. Check logs: `cat ~/.cache/findex-logs/latest.log`

### Hotkey not working

- On Wayland: Bind the toggle command in your compositor settings
- On X11: Check `toggle_key` syntax in `settings.toml`

### CSS changes not applying

- Ensure CSS syntax is valid
- Close and reopen Findex completely
- Check for syntax errors in the file
